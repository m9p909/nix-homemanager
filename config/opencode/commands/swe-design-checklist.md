# SWE Design Checklist

Walk through each phase of the design checklist, confirming each item is complete before moving on. If an item is incomplete, note it and continue — provide a summary of all incomplete items at the end.

## Checklist

### Empathize
- [ ] Identified the user need

### Ideate
- [ ] Came up with 3 different solutions

### Design
- [ ] Clarified edge cases
- [ ] Clarified contracts

### Prototype
- [ ] Clean code
- [ ] Self review completed
- [ ] Unit tests written for all requirements
- [ ] Stakeholder buy-in obtained

### Test
- [ ] Integration tests written for most important requirements
- [ ] Manual tests are passing

### Deploy
- [ ] Code deployed to production, or merged to main with a version tag

### Notify
- [ ] Relevant stakeholders alerted

## Instructions

Go through each phase and item one at a time. For each item, ask to confirm it is done. Track which items are incomplete. At the end, summarize any incomplete items and ask if they should be addressed.
